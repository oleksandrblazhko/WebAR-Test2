# Person
Ти участник WebAR-проекту, який розташовано у поточному каталогу.
Для проведення експериментів було створено тест-програму.
Тест-програма повинна демонструвати основні етапи роботи технології WebAR (AR.js).
Але тест-програма не містить реальний код, який реалізовано в основній програмі проєкту
Це прозводить до проблеми якості візуалізації об'єктів, наприклад, 3D-примітивів, на маркерах.
Наприклад, на сцені 2 розпізнавання контурів гарно працює лише при відповідному куті нахилу вебкамери.
Коли камера нахиляється, багато маркерів вже не розпізнаються.
Тому треба об'єднати код основної програми та тест-програми:
1) залишити всі сцени з тест-програми з урахуванням основних концепцій їх використання;
2) додати до сцен програмний код з основної програми проєкту, щоб забезпечити якість роботи відповідно до основної програми.
Зараз сцена 6 повинна візуалізувати об'єкти на маркерах, тому за своїм призначенням вона відповідає всьому програмному коду з основновної програми.

# Tasks
1) проаналізуй тест-програму для визначення основних кроків алгоритмів у сцені 2;
2) знайди в основній програмі та в інших пов'язаних з нею файлах AR-технології програмний код, який відповідає алгоритму сцени 2;
3) враховуй лише barcode-маркери;
4) не враховуй pattern-маркери та aruco-маркери;
5) надай план перетворення програмного коду для сцени 2;
6) онови програмний код тест-програми для сцени 2 у відповідності з алгоритмами основної програми.

# Context

2) тест-програма - AR-debug-2.html
