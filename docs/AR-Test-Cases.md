### План проведення експериментального тестування (динамічного аналізу)

**Мета:** Оцінити якість та стабільність відстеження barcode-маркерів у різних практичних умовах, щоб виявити сильні та слабкі сторони поточної реалізації.

**Необхідне обладнання:**
1.  **Пристрої:**
    *   Смартфон (для тестування мобільного досвіду).
    *   ПК/ноутбук з веб-камерою.
2.  **Маркери:** Кілька роздрукованих barcode-маркерів (напр., з ID 1, 2, 3) у високій якості. Бажано на матовому папері, щоб уникнути відблисків.
3.  **Оточення:** Кімната з можливістю змінювати освітлення.

---

### Методологія та тест-кейси

Для кожного тесту фіксуйте результати за простою шкалою: **Відмінно** (працює бездоганно), **Добре** (працює, але є незначні артефакти), **Задовільно** (працює нестабільно), **Погано** (майже не працює), **Відмова** (не працює).

#### Тест 1: Стійкість до часткового перекриття

**Мета:** Визначити, наскільки добре система тримає відстеження, коли частина маркера невидима.

**Кроки:**
1.  Розмістіть маркер на столі, досягніть стабільного розпізнавання.
2.  Повільно перекривайте маркер аркушем паперу або пальцем, починаючи з краю. Зафіксуйте, коли відстеження втрачається.
3.  Повторіть, перекриваючи:
    *   ~25% площі (чверть).
    *   ~50% площі (половину).
    *   Один з кутів.
    *   Центральну частину.

**Що спостерігати:**
*   Чи продовжує модель відображатися?
*   Чи починає модель "тремтіти" або зміщуватися?
*   При якому відсотку перекриття відстеження повністю зникає?

#### Тест 2: Реакція на швидкість руху

**Мета:** Оцінити плавність та стабільність 3D-моделі при русі.

**Кроки:**
1.  Зафіксуйте стабільне розпізнавання.
2.  **Рух камери:**
    *   Повільно рухайте камеру вліво-вправо.
    *   Швидко рухайте камеру вліво-вправо (імітація повороту голови).
    *   Повільно наближайте/віддаляйте камеру.
3.  **Рух маркера:**
    *   Зафіксуйте камеру нерухомо.
    *   Повільно рухайте аркуш з маркером.

**Що спостерігати:**
*   **"Тремтіння" (Jitter):** Чи залишається модель стабільно "приклеєною" до маркера, чи вона тремтить?
*   **Затримка (Latency):** Наскільки швидко модель реагує на рух? Чи є видиме "запізнення"?
*   **Втрата відстеження:** Чи зникає модель при швидкому русі?

**Аналіз діагностичних даних (консоль браузера):**
Після додавання діагностичного коду у файл `index.html` (Варіант 2), у консолі браузера ви побачите структуровані повідомлення для маркера з ID 1. Кожне повідомлення міститиме:
*   `Видимість`: Статус розпізнавання маркера.
*   `Час кадру (ms)`: Тривалість обробки поточного кадру.
*   `Pos Δ (Jitter)`: Довжина вектора зміни позиції 3D-об'єкта між поточним та попереднім кадром.
    *   **Інтерпретація:** Це ключовий показник "тремтіння". Якщо при нерухомій камері це значення велике (наприклад, > 0.001-0.005), це вказує на нестабільне відстеження. При плавному русі це значення має бути стабільним і пропорційним швидкості руху. Великі, хаотичні стрибки вказують на сильний "jitter".
*   `Rot Δ (°/кадр)`: Зміна обертання 3D-об'єкта між кадрами у градусах.
    *   **Інтерпретація:** Аналогічно `Pos Δ`, це показник "тремтіння" обертання. Високі значення при нерухомій камері або хаотичні зміни під час руху свідчать про нестабільність обертання.

Коли маркер втрачається з поля зору камери, ви побачите повідомлення `[Тест 2] Маркер X втрачено`.
    

#### Тест 3: Вплив умов освітлення

**Мета:** Перевірити надійність розпізнавання при різній якості та інтенсивності світла.

**Кроки:**
1.  **Ідеальні умови:** Проведіть тест при яскравому, розсіяному денному світлі. Це буде ваш базовий показник.
2.  **Слабке освітлення:** Проведіть тест ввечері при кімнатному освітленні.
3.  **Пряме світло / Відблиски:** Спрямуйте на маркер світло від лампи або ліхтарика так, щоб з'явилися відблиски.

**Що спостерігати:**
*   Чи розпізнається маркер взагалі?
*   Наскільки стабільним є відстеження порівняно з ідеальними умовами?

#### Тест 4: Вплив кута огляду

**Мета:** Визначити, під яким гострим кутом до камери маркер ще розпізнається.

**Кроки:**
1.  Розпочніть з перпендикулярного положення маркера до камери (кут ~90°).
2.  Повільно нахиляйте маркер від себе, зменшуючи кут.
3.  Зафіксуйте приблизний кут, при якому відстеження стає нестабільним або зникає.

**Що спостерігати:**
*   Граничний кут (напр., 45°, 30°, 20°).
*   Чи починає модель спотворюватися або "стрибати" перед втратою відстеження?

### Формат звітування

Пропоную створити просту таблицю для фіксації результатів, наприклад, у файлі `docs/Test-Results.md`:

```markdown
# Результати динамічного аналізу barcode-маркерів

| Тест-кейс                      | Умови тесту                               | Результат (Оцінка) | Коментарі                                             |
|--------------------------------|-------------------------------------------|--------------------|-------------------------------------------------------|
| **1. Часткове перекриття**     | Перекриття краю (~25%)                    | Добре              | Модель трохи зміщується, але відстеження стабільне.    |
|                                | Перекриття центру (~25%)                  | Задовільно         | Відстеження періодично втрачається.                    |
| **2. Швидкість руху**          | Повільний рух камери                      | Відмінно           | Модель рухається плавно.                              |
|                                | Швидкий рух камери                        | Задовільно         | Спостерігається сильне тремтіння та втрата відстеження.|
| **3. Освітлення**              | Слабке кімнатне світло                    | Погано             | Маркер розпізнається лише зблизька.                    |
|                                | Відблиски на маркері                      | Відмова            | Відстеження неможливе.                                |
| **4. Кут огляду**              | Нахил до ~45°                             | Відмінно           | Без проблем.                                          |
|                                | Нахил до ~30°                             | Задовільно         | Починається тремтіння, можлива втрата відстеження.     |
```

Проведення цих тестів дасть вам об'єктивні дані, на основі яких можна буде:
1.  Порівняти ефективність barcode-маркерів з Aruco в майбутньому.
2.  Сформулювати рекомендації для кінцевих користувачів (напр., "використовуйте в добре освітлених приміщеннях").
3.  Визначити пріоритети для подальших технічних покращень.
