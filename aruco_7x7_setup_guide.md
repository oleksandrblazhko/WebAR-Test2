# Використання 7x7 ArUco маркерів у вашому проекті WebAR

## Огляд

Ваш проект WebAR вже підтримує ArUco маркери, і тепер ви можете використовувати 7x7 ArUco маркери для більш надійного розпізнавання. 7x7 маркери мають більше унікальних комбінацій і кращу стійкість до шуму порівняно з 4x4 маркерами.

## Основні переваги 7x7 маркерів

- **Більше унікальних маркерів**: 1000 унікальних маркерів у словнику ARUCO_7X7_1000
- **Краща точність**: Більша кількість бітів даних забезпечує краще розпізнавання
- **Надійність**: Краще працюють при нижчій якості зображення або при нахилах

## Як створити 7x7 маркер

1. Відкрийте файл `aruco-marker-generator.html` у браузері
2. Виберіть "ARUCO 7x7 (1000 маркерів)" зі списку словників
3. Введіть ID маркера (від 0 до 999)
4. Натисніть "Згенерувати маркер"
5. Завантажте SVG-файл та надрукуйте маркер

## Як використовувати 7x7 маркер у вашому проекті

### 1. Оновлення конфігурації

Щоб використовувати 7x7 маркер, вам потрібно оновити файл `config.json`. Додайте маркер типу "aruco" з вказівкою словника:

```json
{
  "markers": [
    {
      "type": "aruco",
      "id": 0,
      "dictionary": "ARUCO_7X7_1000",
      "content": {
        "type": "model",
        "file": "model.glb"
      }
    }
  ]
}
```

### 2. Альтернативний спосіб (для існуючих налаштувань)

Якщо у вас вже є налаштовані ARUCO маркери, ви можете просто додати параметр словника до існуючих налаштувань:

```json
{
  "markers": [
    {
      "type": "aruco",
      "id": 0,
      "dictionary": "ARUCO_7X7_1000",  // Додайте цей рядок для використання 7x7
      "content": {
        "type": "model",
        "file": "Bender.glb"
      }
    }
  ]
}
```

### 3. Підключення словника у коді

У файлі `index.html` вже підключено файл `aruco_4x4_1000_browser.js`, але для 7x7 маркерів вам потрібно буде додати:

```html
<script src="js-aruco2/dictionaries/aruco_7x7_1000.js"></script>
```

## Приклади використання

### Приклад 1: Налаштування простого 7x7 маркера

```json
{
  "markers": [
    {
      "type": "aruco",
      "id": 5,
      "dictionary": "ARUCO_7X7_1000",
      "content": {
        "type": "model",
        "file": "kopatuch.glb",
        "scale": 0.1
      }
    }
  ]
}
```

### Приклад 2: Налаштування 7x7 маркера з альтернативним вмістом

```json
{
  "markers": [
    {
      "type": "aruco",
      "id": 10,
      "dictionary": "ARUCO_7X7_1000",
      "content": {
        "type": "model",
        "file": "Octopus_toy.glb"
      },
      "alternativeContent": {
        "contentType": "primitive",
        "primitive": {
          "type": "ConeGeometry",
          "radius": 0.5,
          "height": 1,
          "color": "#ff0000",
          "position": [0, 0.5, 0],
          "animation": {
            "property": "rotation.y",
            "speed": 0.02
          }
        }
      }
    }
  ]
}
```

## Порівняння розмірів маркерів

| Тип маркера | Розмір | Унікальні маркери | Використання |
|-------------|--------|-------------------|--------------|
| ARUCO (4x4) | 4x4 пікселі | 256 | Для простих застосункань |
| ARUCO_4X4_1000 | 4x4 пікселі | 1000 | Для більшої кількості унікальних маркерів |
| ARUCO_7X7_1000 | 7x7 пікселі | 1000 | Для кращої точності та надійності |

## Поради щодо використання

1. **Розмір маркера**: 7x7 маркери потребують більше пікселів на екрані для надійного розпізнавання
2. **Освітлення**: Переконайтесь, що маркер добре освітлений без затемнень
3. **Якість друку**: Для найкращих результатів друкуйте маркери на високоякісному принтері
4. **Перевірка**: Завжди тестуйте маркер перед використанням у фінальному застосунку

## Налагодження

Якщо 7x7 маркер не розпізнається:

1. Переконайтесь, що словник правильно вказаний у `config.json`
2. Перевірте, чи файл `aruco_7x7_1000.js` підключений у вашому HTML
3. Переконайтесь, що ID маркера відповідає згенерованому
4. Перевірте якість друку та освітлення при скануванні
