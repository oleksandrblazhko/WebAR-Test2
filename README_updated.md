# WebAR Board Game Project

Цей проект є шаблоном для створення WebAR (Web Augmented Reality) ігор та додатків з використанням технологій ARToolkit, Three.js та js-aruco2.

## Особливості

- Підтримка різних типів маркерів: шаблонні (pattern), штрих-коди (barcode) та ArUco маркери
- Підтримка 5x5 та 7x7 ArUco маркерів для кращого розпізнавання
- Вбудована система аудіо та відео відтворення
- Підтримка 3D моделей у форматах GLB/GLTF
- Система контролерів для взаємодії з об'єктами
- Альтернативний вміст для кожного маркера

## Нова функція: 5x5 та 7x7 ArUco маркери

Тепер проект підтримує 5x5 та 7x7 ArUco маркери, які пропонують:

- **Більше унікальних маркерів**: До 1000 унікальних маркерів у словниках ARUCO_5X5_1000 та ARUCO_7X7_1000
- **Кращу точність**: Більша кількість бітів даних забезпечує краще розпізнавання
- **Надійність**: Краще працюють при нижчій якості зображення або при нахилах
- **Баланс**: 5x5 маркери пропонують баланс між кількістю унікальних маркерів та точністю розпізнавання

### Як використовувати 5x5 та 7x7 ArUco маркери

#### 5x5 ArUco маркери

1. **Створіть маркер**:
   - Відкрийте `aruco-marker-generator.html` у браузері
   - Виберіть "ARUCO 5x5 (1000 маркерів)" зі списку
   - Введіть ID маркера (від 0 до 999)
   - Натисніть "Згенерувати маркер" та завантажте SVG-файл

2. **Налаштуйте конфігурацію**:
   ```json
   {
     "id": 13,
     "type": "aruco",
     "dictionary": "ARUCO_5X5_1000",
     "markerId": 5,
     "contentType": "model",
     "contentFile": "Octopus_toy.glb"
   }
   ```

#### 7x7 ArUco маркери

1. **Створіть маркер**:
   - Відкрийте `aruco-marker-generator.html` у браузері
   - Виберіть "ARUCO 7x7 (1000 маркерів)" зі списку
   - Введіть ID маркера (від 0 до 999)
   - Натисніть "Згенерувати маркер" та завантажте SVG-файл

2. **Налаштуйте конфігурацію**:
   ```json
   {
     "id": 14,
     "type": "aruco",
     "dictionary": "ARUCO_7X7_1000",
     "markerId": 10,
     "contentType": "model",
     "contentFile": "Octopus_toy.glb"
   }
   ```

3. **Додайте до config.json**:
   Додайте новий об'єкт маркера до масиву "markers" у вашому файлі config.json

## Структура проекту

- `index.html` - Головний HTML файл
- `config.json` - Конфігурація маркерів та вмісту
- `js/` - Користувацькі JavaScript файли
- `js-aruco2/` - Бібліотека для роботи з ArUco маркерами
- `jsartoolkit5/` - Бібліотека ARToolkit
- `threex/` - Розширення для Three.js
- `loaders/` - Завантажувачі для різних форматів файлів
- `textures/`, `data/` - Медіафайли для проекту

## Типи маркерів

### Pattern маркери
- Використовують файл `.patt` для розпізнавання
- Підходять для складних зображень
- Приклад: `pattern-ScanMe.patt`

### Barcode маркери
- Розпізнають числові ID
- Прості в реалізації
- Приклад: `"target": 1` для штрих-коду з ID 1

### ArUco маркери
- Квадратні маркери з унікальними шаблонами
- Висока точність розпізнавання
- Підтримуються 4x4, 5x5 та 7x7 розміри

## Типи вмісту

- `model` - 3D модель (GLB/GLTF)
- `primitive` - Примітивна геометрія (куб, сфера тощо)
- `video` - Відеофайл
- `image` - Зображення
- `text` - Текстова геометрія
- `controller` - Контролер для взаємодії

## Налаштування анімації

Будь-який об'єкт може мати анімацію, вказавши параметр `animation`:

```json
"animation": {
  "type": "rotation",
  "axis": "y",
  "speed": 0.01
}
```

## Альтернативний вміст

Кожен маркер може мати альтернативний вміст, який активується при натисканні на екран:

```json
"alternativeContent": {
  "contentType": "primitive",
  "primitive": {
    "type": "ConeGeometry",
    "radius": 0.5,
    "height": 1,
    "color": "#ff0000"
  }
}
```

## Запуск проекту

Проект можна запустити локально за допомогою сервера, наприклад:

- Python: `python -m http.server 8000`
- Node.js: `npx serve .`
- PHP: `php -S localhost:8000`

Потім відкрийте `http://localhost:8000` у браузері з підтримкою WebRTC (Chrome, Firefox, Edge).
